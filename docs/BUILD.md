# Build Guide

## Prerequisites

| Tool | Minimum version | Check |
|---|---|---|
| Node.js | 18.x | `node -v` |
| npm | 9.x | `npm -v` |

All other tools (TypeScript, Turbo, tsup, ng-packagr, Vite) are installed locally as workspace devDependencies — no global installs required.

---

## First-time setup

```bash
git clone <repo-url>
cd fs-design-system
npm install          # installs all workspace packages in one pass
```

npm workspaces hoist shared dependencies to the root `node_modules/` and symlinks each `packages/*` and `apps/*` into it so cross-package imports resolve without publishing.

---

## Building everything

```bash
npm run build
```

Turbo runs the `build` pipeline respecting the dependency graph:

```
@fs-ds/tokens
    └─ @fs-ds/web-components
           ├─ @fs-ds/react
           ├─ @fs-ds/angular
           ├─ @fs-ds/demo-react
           ├─ @fs-ds/demo-angular
           ├─ @fs-ds/demo-html
           └─ @fs-ds/storybook (build-storybook)
```

Turbo caches each task's output. Subsequent runs only rebuild packages whose source has changed.

---

## Building individual packages

```bash
# From the repo root — filter by package name
npx turbo run build --filter=@fs-ds/tokens
npx turbo run build --filter=@fs-ds/react
npx turbo run build --filter=@fs-ds/web-components
npx turbo run build --filter=@fs-ds/angular

# Or cd into the package directly
cd packages/tokens && npm run build
```

## Package-by-package details

### `@fs-ds/tokens`

**Tool:** `tsup` + plain Node.js copy script

```bash
cd packages/tokens
npm run build
```

Outputs:
```
dist/
├── index.js       CJS bundle
├── index.mjs      ESM bundle
├── index.d.ts     TypeScript declarations
├── index.d.mts    ESM declarations
└── tokens.css     CSS custom properties (copied from src/)
```

The `tokens.css` file is the most commonly consumed artifact.  
Consumers import it once at the app entry point:
```ts
import '@fs-ds/tokens/css';
```

---

### `@fs-ds/web-components`

**Tool:** `tsc` (TypeScript compiler)

```bash
cd packages/web-components
npm run build
```

Outputs:
```
dist/
├── index.js / index.d.ts
├── shared/base-styles.js
└── components/
    ├── button/fs-button.js + .d.ts
    ├── input/fs-input.js  + .d.ts
    └── badge/fs-badge.js  + .d.ts
```

Key tsconfig flags:
- `experimentalDecorators: true` — required by Lit decorators (`@customElement`, `@property`)
- `useDefineForClassFields: false` — required by Lit's property system

---

### `@fs-ds/react`

**Tool:** `tsup` (CJS + ESM + DTS)

```bash
cd packages/react
npm run build
```

Outputs:
```
dist/
├── index.js      CJS
├── index.mjs     ESM
├── index.d.ts    declarations (CJS)
└── index.d.mts   declarations (ESM)
```

All workspace deps (`@fs-ds/web-components`, `@fs-ds/tokens`, `lit`) and peer deps (`react`, `react-dom`) are marked `--external` so they are **not** bundled — the consuming app provides them.

---

### `@fs-ds/angular`

**Tool:** `ng-packagr` (Angular library format)

```bash
cd packages/angular
npm run build
```

Outputs the Angular Package Format (APF):
```
dist/
├── esm2022/         ES2022 FESM bundle
├── fesm2022/        Flat ESM bundle (tree-shakeable)
├── index.d.ts       declarations
└── package.json     generated by ng-packagr
```

`@fs-ds/tokens` and `@fs-ds/web-components` are `peerDependencies` — the consuming Angular app must install them separately.

## Development watch mode

Run all packages in watch mode in parallel:

```bash
npm run dev
```

Or target a single package:

```bash
cd packages/tokens && npm run dev
cd packages/react  && npm run dev
```

Watch mode for `@fs-ds/angular`:

```bash
cd packages/angular && npm run dev   # ng-packagr --watch
```

## Clean all build artifacts

```bash
npm run clean
```

This removes every `dist/`, `.next/`, `storybook-static/`, and `node_modules/.cache` across all workspaces via Turbo.

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| `Cannot find module '@fs-ds/tokens'` | Package not built yet | `npm run build --filter=@fs-ds/tokens` |
| `@customElement is not a decorator` | Wrong `tsconfig` flags | Ensure `experimentalDecorators: true` and `useDefineForClassFields: false` in web-components tsconfig |
| Turbo cache stale | Source changed but hash not updated | `npx turbo run build --force` |
| ng-packagr `allowedNonPeerDependencies` error | Dependency in `dependencies` instead of `peerDependencies` | Move to `peerDependencies` in angular `package.json` |
